# EduFlow Platform v3.0 发布说明

## 📅 发布日期
2026-04-26

## 🎯 版本概述

EduFlow Platform v3.0 是一次重大更新，引入了全新的 Vue3 前端架构、优化的 AI 智能助手、统一的访问入口，以及全面的 Docker 部署优化。

---

## ✨ 主要新特性

### 1. 全新 Vue3 前端架构
- **技术栈升级**：从 Bootstrap/jQuery 升级到 Vue 3.4 + Element Plus 2.6
- **现代化构建**：使用 Vite 5.2 快速构建工具
- **组件化开发**：完整的 Vue3 组件体系
- **响应式设计**：Element Plus 企业级 UI 组件
- **状态管理**：Pinia 2.1 现代化状态管理
- **路由管理**：Vue Router 4.3 前端路由

### 2. AI 智能助手优化
- **自然对话风格**：优化 Prompt 设计，避免文章化输出
- **连续对话支持**：支持上下文记忆和多轮对话
- **多模型切换**：DeepSeek、豆包、GLM、OpenAI 多提供商支持
- **免登录访问**：AI 接口添加 `@NoNeedLogin` 注解
- **统一访问入口**：通过 80 端口统一访问

### 3. 统一访问入口
- **Nginx 反向代理**：80 端口统一代理所有请求
- **智能路由**：
  - 静态资源 → EduFlow Frontend (3006)
  - API 请求 → Spring Boot Backend (8020)
  - AI 接口 → 直接代理，保持路径完整
- **简化访问**：用户只需记住 `http://localhost:80/`

### 4. Docker 全面优化
- **多阶段构建**：减少镜像体积，提高构建效率
- **环境变量配置**：通过 `.env` 文件灵活配置
- **服务编排**：docker-compose 管理所有服务
- **健康检查**：自动监控服务状态
- **自动重启**：服务异常自动恢复

## 🐛 问题修复

### 已修复的问题

1. **AI 接口被拦截器拦截**
   - 问题：所有 `/api/ai/**` 请求被登录拦截器阻止
   - 修复：添加 `@NoNeedLogin` 注解和路径白名单

2. **Nginx 代理路径重写导致 404**
   - 问题：`/api/ai/create` 被重写为 `/ai/create`，后端路径不匹配
   - 修复：为 AI 接口添加单独的 location 规则，不走重写

3. **前后端响应格式不一致**
   - 问题：前端检查 `code === 200`（数字），后端返回 `code: "0000"`（字符串）
   - 修复：前端改为字符串比较 `code === '0000'`

4. **流式对话页面连接失败**
   - 问题：页面尝试连接不存在的 WebSocket 端点
   - 修复：移除流式对话页面重定向，统一使用 EduFlow Platform

5. **Docker 构建 Maven 镜像 SSL 握手失败**
   - 问题：阿里云 Maven 镜像在 Docker 容器内 SSL 不稳定
   - 修复：切换为华为云镜像，添加 SSL 宽容配置

## 🔧 技术改进

### 代码质量改进

1. **API 响应格式统一**
   ```json
   {
     "code": "0000",
     "data": "...",
     "message": "操作成功"
   }
   ```

2. **Prompt 设计优化**
   - System Prompt：从"内容创作助手"改为"友好、专业的AI助手"
   - 用户 Prompt：从"生成文章"改为"自然对话方式回答"
   - 新增 `generateChatPrompt()` 方法，专门用于对话场景

3. **Nginx 配置优化**
   ```nginx
   # AI 接口代理（不走重写）
   location /api/ai/ {
       proxy_pass http://absadmin_backend/api/ai/;
   }
   
   # 其他 API 代理（去掉 /api 前缀）
   location /api/ {
       rewrite ^/api/(.*)$ /$1 break;
       proxy_pass http://absadmin_backend;
   }
   ```

4. **Maven 构建优化**
   - 华为云镜像作为主镜像
   - 添加 SSL 宽容配置
   - 本地缓存策略

## 🚀 部署指南

### 快速部署（Docker Compose）

```bash
# 1. 克隆项目
git clone https://github.com/cruise991/EduFlow-Platform.git
cd EduFlow-Platform

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，配置数据库和 AI API Keys

# 3. 启动所有服务
docker-compose up -d --build

# 4. 访问应用
# 统一入口：http://localhost:80/
# 备用入口：http://localhost:3006/
```

### 本地开发

#### 后端开发
```bash
mvn clean package -DskipTests
java -jar target/EduFlow-Platform.jar
```

#### 前端开发
```bash
cd frontend
npm install
npm run dev
```

## 📊 系统架构

### 服务架构

```
┌─────────────────────────────────────────────┐
│              Nginx (80, 443)                │
│         统一入口、反向代理、负载均衡         │
└───────────────┬─────────────────────────────┘
                │
        ┌───────┴───────┐
        │               │
   静态资源         API 请求
        │               │
        ↓               ↓
┌──────────────┐  ┌──────────────┐
│ Vue3 Frontend│  │Spring Boot   │
│  (3006)      │  │  (8020)      │
│              │  │              │
│ - Element    │  │ - RESTful    │
│   Plus       │  │   API        │
│ - Vue Router │  │ - MyBatis    │
│ - Pinia      │  │ - AI Service │
└──────────────┘  └──────┬───────┘
                         │
                  ┌──────┴──────┐
                  │             │
             业务逻辑      外部服务
                  │             │
                  ↓             ↓
           ┌──────────┐  ┌──────────
           │  MySQL   │  │ DeepSeek │
           │  (3306)  │  │ AI API   │
           └──────────┘  └──────────
```

### 核心组件

| 组件 | 容器名 | 端口 | 技术栈 | 说明 |
|------|--------|------|--------|------|
| Nginx | absadmin-nginx | 80, 443 | Nginx Alpine | 反向代理、静态资源 |
| Frontend | eduflow-frontend | 3006 | Vue3 + Element Plus | 前端应用 |
| Backend | absadmin-app | 8020 | Spring Boot 3.2.5 | 后端 API 服务 |
| Database | absadmin-mysql | 3306 | MySQL 8.0 | 关系型数据库 |

## 🎯 使用指南

### 访问方式

#### 统一入口（推荐）
- URL：`http://localhost:80/`
- 说明：通过 Nginx 统一代理，所有请求由 80 端口进入

## 📝 已知问题

### 当前限制

1. **前端构建产物未独立打包**
   - 现状：前端构建产物包含在后端 JAR 中
   - 影响：后端 JAR 体积较大（444MB）
   - 计划：未来版本将分离前后端打包

2. **WebSocket 流式输出未实现**
   - 现状：使用 RESTful API 一次性返回
   - 影响：长文本回复需要等待完整生成
   - 计划：未来实现 SSE 流式输出

3. **HTTPS 未配置**
   - 现状：仅支持 HTTP
   - 影响：生产环境需要手动配置 SSL
   - 计划：提供 HTTPS 配置示例
